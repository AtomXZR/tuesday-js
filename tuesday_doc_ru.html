TuesDay

visual novel engine

Tuesday бесплатный движок с открытым исходным кодом для создания визуальных новелл, интерактивных историй и текстовых приключенческих игр. Написан на языке javaScript без использования сторонних библиотек. Присутствует поддержка инфраструктуры Apache Cordova для создания нативных приложений для мобильных устройств и настольных компьютеров. Движок использует стандартные элементы Html документа, как div и img, что делает возможным использование почти любых медиа-форматов поддерживаемых браузерами, включая векторную графику в формате svg, gif анимации и обширные возможности css стилей.

Структура


Проект состоит из 3-х основных элементов. Html файла (index.html) с подключенным к нему файлом движка tuesday.js и сценарией истории в json файле или let массиве. Отображение сценария на html странице происходит через элемент div с id="tuesday". Параметры отображения можно задать в стиле css или любым другим подходящим способом. Все содержание будет отображаться с учетом параметров этого элемента.


Сценарий истории со всеми элементами указывается в структуре формата JSON. Этот формат может быть прочитан практически любым другим языком программирования или движком. (что дает возможность при необходимости перенести историю на другой движок)


Сценарий запускается функцией load_story( 'тип', файл / массив ) с указанием типа данных (data или file) и их источник (адрес файла или массив)


data: указание переменной с массивом сценаря из кода javaScript

file: путь к json файлу со сценарием (может не работать локально, если в браузере включена cross domain защита)

AutoSave


Параметр отвечает за автоматическое сохранение прогресса сценария и значений из массива variables на каждом диалоге. Это позволит продолжить историю с момента завершения последней сессии.

	"autosave": настройка автосохранения прогресса true / false,

За загрузку автосохранения отвечает атрибут "tue_load_autosave" указанный в "name" или "go_to"

Languares


Массив поддерживаемых языков проекта, по умолчанию установлен английский "en". Движок на старте автоматически определяет язык браузера и при наличии подходящей локализации автоматически переключается на нее. 	"languares": [коды языков доступных в проекте],

Все коды языков можно узнать на: "HTML Language Code Reference"


Working example "languares"

Сценарий истории


Структура сценария разделена на несколько элементов. Основным элементом является "блок" это часть повествования или сюжетная линия, и их последовательность будет определяться выборами пользователя (выбор идти налево или направо может означать, что игрок продолжено продолжить текущий блок сюжета или перейдет к другому блоку с альтернативным развития сюжетом). Блоки состоят из сцен с набором графических и звуковых элементов. И уже внутри сцены указывается последовательность диалогов и других элементов как выбор "choice" и "go_to", которые могут привести к другому блоку.


"Блок 1": [
	{
		Сцена 1:[
			{Диалог 1},
			{Диалог 2},
			{Диалог 3}
		]
	},{
		Сцена 2:[
			{Диалог 1},
			{Диалог 2},
			{Диалог 3}
		]
	}
],
Блок 2": [
	{	
		Сцена 1:[
			{Диалог 1},
			{Диалог 2},
			{Диалог 3}
		]
	},{
		Сцена 2:[
			{Диалог 1},
			{Диалог 2},
			{Диалог 3}
		]
	}
]

Элементы сцены


Background music


Запускает зацикленную фоновую музыку в сцене.

	"background_music": "путь к аудио файлу без расширения только его имя"

Поддерживается локализация

	"background_music": {
			"en":"путь к аудио файлу для английского языка",
			"ja":"путь к аудио файлу для японского языка",
			"ru":"путь к аудио файлу для русского языка"
		}



Background image


Аналог параметра css background_image. Предназначен для отображения заднего фона в сцене.

	"background_image": "путь к файлу с изображением"

По умолчанию выравнивается по центру сцены и пропорционально заполняет весь фон, независимо от размера сцены или самого файла.


Поддерживает локализацию если есть фон для конкретного перевода.

	"background_image": {
			"en":"путь к файлу с изображением для английского языка",
			"ja":"путь к файлу с изображением для японского языка",
			"ru":"путь к файлу с изображением для русского языка"
		}



Dialogs


Массив последовательности диалогов в сцене и ее атрибутов, таких как art и кнопки выбора Choice.

	"dialogs":[
			{Параметры и элементы реплики 1й диалога},
			{Параметры и элементы реплики 2й диалога},
			{Параметры и элементы реплики 3й диалога}
	]





Элементы диалога


Art


Онже "sprite" или "image". Аналог html тега <img>. Предназначен для отображения графических элементов в сцене, такие как персонажи или предметы.

"art": [{		
		"url":"путь к файлу с изображением",
		"position": [позиция X, позиция Y],
		"size": ["ширина", "высота"],
		"style": "параметры css style и их значения",
		"className": "набор классов стилей css"
	}]

Поддерживает локализацию если есть изображения созданные для конкретного перевода указанного в разделе "parameters" / "languares".

		"url": {
				"en":"путь к файлу с изображением для английского языка",
				"ja":"путь к файлу с изображением для японского языка",
				"ru":"путь к файлу с изображением для русского языка"
			}

Можно добавить в сцену несколько изображений, создав в массиве дополнительные элементы в фигурных скобках со всеми необходимыми параметрами.

"art": [{		
		"url":"Изображени 1",
		"position": [позиция X, позиция Y],
		"size": ["ширина", "высота"]
	},{		
		"url":"Изображени 2",
		"position": [позиция X, позиция Y],
		"size": ["ширина", "высота"]
	}]

Working example "art"

"art": [{		
		"position": ["75%", "50%"],
		"size": ["auto", "90%"],
		"style": "transform: translateX(-50%) translateY(-50%);",
		"className": "example",
		"url": {
				"en":"img/example_en.png",
				"ja":"img/example_ja.png",
				"ru":"img/example_ru.png"
		}
	},{		
		"position": ["25%", "50%"],
		"size": ["auto", "90%"],
		"style": "transform: translateX(-50%) translateY(-50%);",
		"url":"img/character.png"
	}]



Text


Диалог персонажа или иные тексты в конкретной сцене. А также элементы для изменения text_panel" на данном тексте.

	"text": {
		"art":"путь на фоновую картинку панели",
		"art_size": ["ширина", "высота"] изображения на кнопке,
		"className": "набор классов стилей css",
		"color_text": "цвет текста",
		"color": "цвет в форматах",
		"indent_bottom": "отступ текстовой панели от нижнего края",
		"position": [лево, право, верх, низ] позиция отнасительна краев сцены, 
		"size": ["ширина", "высота"],
		"size_text": "Размер текста",
		"style": "параметры css style и их значения"
		"en":"Boss"
	}

Можно просто указать реплику, в этом случае все недостающие параметры будут взяты по умолчанию из раздела "parameters" / "text_panel"

	"text": "Hello how are you"

Возможно добавить локализованный вариант диалога для указанных языков в разделе "parameters" / "languares"

	"text": {
		"en":"Hello how are you",
		"ja":"こんにちは、元気ですか",
		"ru":"Привет как дела"
		}

В диалоговом окне могут отображаться значения переменной из раздела "parameters" / "variables". Если вы укажете имя переменной в угловых скобках , то вместо этого в тексте появится значение из одноименной переменной. 

{
	"parameters": {
		другие значения
		"variables": [
			"health_points": 100
		],
	}
	"example_block_1": [{
		"dialogs": [{
			"text": "You have <health_points> health points left"
		}]
	}]
}

В этом случае появится фраза "You have 100 health points left"


В одной локации может быть больше или меньше реплик чем в другой. В этом случае можно указать значение 'skip', тогда эта реплика будет автоматически пропущена.

{
	"text": {
		"en":"Hello!",
		"ja":"こんにちは、元気ですか",
		"ru":"Привет!"
	}
},{
	"text": {
		"en":"How are you",
		"ja":"skip",
		"ru":"Kак дела"
	}
}

В этом случае в японской локализации будет пропущена 2-я реплика




Text add


Аналог обычного элемента "text" за исключением того что текст из данного элемента добавляеться к уже существующиму тексту в text_panel". Основной элемен для кинетических новеллы

	"text_add": {
		"art":"путь на фоновую картинку панели",
		"art_size": ["ширина", "высота"] изображения на панеле,
		"className": "набор классов стилей css",
		"color_text": "цвет текста",
		"color": "цвет в форматах",
		"indent_bottom": "отступ текстовой панели от нижнего края",
		"position": [лево, право, верх, низ] позиция относительно краев сцены, 
		"size": ["ширина", "высота"],
		"size_text": "Размер текста",
		"style": "параметры css style и их значения"
		"en":"Boss"
	}

Working example "variables"

{
	"text": "Hello! "
},{
	"text_add": "will you go for a walk?"
}
В текстовой панеле сначала появиться "Hello!" затем вся фраза целиком "Hello! will you go for a walk?"



Name


Параметр для отображения имени и аватара персонажа в текстовой панели. Эти данные можно указать несколькими способами.


Самый простой вариант просто указать его имя. Если это имя не совпадает с id идентификатором в разделе "parameters" / "characters", тогда имя отобразится так как указанно в этом параметре. 

	"name": "Boss"

Возможно задать локализованный вариант имени для указанных языков в разделе "parameters" / "languares"

	"name": {
		"en":"Boss",
		"ja":"ボス",
		"ru":"Босс"
		}

Если вы часто используете одно и то же имя, вы можете указать его в разделе "parameters" / "characters". Это сильно упрощает процесс смены именами.

{
	"parameters": {
		другие значения
		"characters": {
			"boss": {
				"en":"Boss",
				"ja":"ボス",
				"ru":"Босс"
			}
		}
	}
	"example_block_1": [{
		"dialogs": [{
			"name": "boss"
		}]
	}]
}

Параметры для изменения "name_panel" можно указать как в "dialogs" для текущей реплики, так и в массиве "characters" для конкретного имени.

	"text": {
		"art":"путь на фоновую картинку кнопки",
		"art_size": ["ширина", "высота"] изображения на панеле,
		"className": "набор классов стилей css",
		"color_text": "Цвет текста",
		"color": "цвет элемента",
		"indent_bottom": "отступ текстовой панели от нижнего края",
		"position": [лево, право, верх, низ] позиция отнасительна краев сцены, 
		"size": ["ширина", "высота"],
		"size_text": "Размер текста",
		"style": "параметры css style и их значения",
		"en":"Boss"
	}

Если какой-то элемент будет не указан, то будет взято значение по умолчанию из "parameters" / "name_panel". Если имени или его id идентификатора будут отсутствовать, то эта панель не появиться в сцене.





Choice


Кнопки для выбора дальнейшего сюжетного блока и изменения значений в массиве переменных "parameters" / "variables". Аналогичен параметру "buttons" за исключением того что кнопки выбора появляются в определенной реплике. 

	"choice": [{
		"go_to": "id блока для перехода",
		"position": [позиция X, позиция Y],
		"size": ["ширина", "высота"],
		"className": "набор классов стилей css",
		"style": "параметры css style и их значения",
		"sound": "Id звука"
		"sound_stop": "Id звука для остановки"
		"color": "цвет фона",
		"color_text": "Цвет текста",
		"variables": [
			["название переменной", "операция с переменной", значение]
		],
		"text": {
			"en":"Перевод текста на английский",
			"ja":"Перевод текста на японский",
			"ru":"Перевод текста на русский"
			}
		}
	]

Парметр "go_to" переносит дальнейшее повествование на указанный блок. Если парметр "go_to" отсутствует, то будет продолжен текущий блок.


Параметры "variables" вносят изменения в переменные в момент нажатия на кнопку выбора.


Working example "choice"

{ "name": "inga",
	"text": "which road to take?"
	"choice": [{
		"go_to": "block_left",
		"position": ["25%", "0", "0", "50%"],
		"size": ["256px", "0"],
		"className": "buttons hot_center",
		"text": "<== to the left"
	},{
		"go_to": "block_right",
		"position": ["75%", "0", "0", "50%"],
		"size": ["256px", "0"],
		"className": "buttons hot_center",
		"text": "on the right ==>"
	}]
}



Variables


Параметр для изменения значений из раздела "parameters" / "variables" в конкретном "dialogs" или при выборе "choice"

	"variables": [
		["название переменной", "операция с переменной", значение]
	],

Операция с переменной

"set" - установить значение в переменной ( аналог оператора "=" в javaScript)

"add" - прибавить значение к уже заданному значению в переменной (аналог оператора "+=" в javaScript)


Working example "variables"

{
    "parameters": {
		другие значения
		"variables": [
			"experience_points": 80,
			"magic_points": 100,
			"health_points": 100
		],
	},
	"example_block_1": [{
		"dialogs": [{
			"variables": [
				["experience_points", "add", 10],
				["magic_points", "add", -15],
				["health_points", "set", 200]
			]
		}]
	}]
}

При запуске блока "example_block_1"
10 очков опыта будут добавлены к значению "Experience_points",
15 очков будет вычтено из "magic_points",
а "health_points" будет установлено на 200 очков.





Events


Элемент предназначенный для внешнего использования в javaScript. Он сигнализирует о начале момента в сюжете где указан данный элемент.


tusday - создание событие 
	"event": "id_event",

javaScript - реагирование на событие 
tuesday.addEventListener(id_event', function(event) { alert("event starting") } );




Legacy choice


Автоматический переход к другому блоку истории при соблюдении указанных условий в переменных значениях ( parameters / variables )

{
	"legacy_choice": [
		["название переменной", "оператор сравнения", значение для сравнения, "id блока для перехода"]
	]
}

Важный! Этот элемент не является частью массива "dialogs". И запускать как часть блока рядом с элементами "dialogs". 

"example_block_1": [{
		"dialogs": [{"text": "hello"}]
	},{
		"legacy_choice": [
			["название переменной", "оператор сравнения", значение для сравнения, "id блока для перехода"]
		]
	}
]

Если условия не совпадают со значениями в переменных то можно указать параметр "go_to", то в этом случае история перейдет в указанный блок.

{
	"legacy_choice": [
		["название переменной", "оператор сравнения", значение для сравнения, "id блока для перехода"],
		{"go_to": "example_block_4"}
	]
}

Working example "legacy_choice"

{
    "parameters": {
		другие значения
		"variables": [
			"experience_points": 80,
			"magic_points": 100,
			"health_points": 100
		],
	},
	"example_block_1": [{
		"legacy_choice": [
			["experience_points", "=", 80, "example_block_2"],
			["experience_points", "<", 80, "example_block_3"],
			{"go_to": "example_block_4"}
		]
	}],
	"example_block_2": [{
		"dialogs": [{
			"text": "Awesome!!!"
		}]
	}],
	"example_block_3": [{
		"dialogs": [{
			"text": "you cool"
		}]
	}],
	"example_block_4": [{
		"dialogs": [{
			"text": "hello"
		}]
	}]
}





Параметры элементов



"art" - Аналог css стиля background_image. указывает фоновый рисунок.

		"art":"путь на файл с изображением",

Поддерживается вариант с локализацией 
	"art": {
		"en":"путь на файл с изображением на английский",
		"ja":"путь на файл с изображением на японский",
		"ru":"путь на файл с изображением на русский"
		}
	}


"art_size" - Устанавливает размер изображения внутри элемента, иначе оно отображаться в исходном размере, Задав размер высоты и ширены 100%, размер изображения будет полностью соответствовать размеру кнопки (параметр size). заменив одно из значений на «авто», тогда размер изображения будет пропорционален размеру указанной высоты или ширины

		"art_size": ["ширина", "высота"] размер изображения внутри элемента,


"size_text" - Размер текста внутри элемента

		"size_text": "Размер текста",


"back_to" - Переход по нажатию кнопки назад (tue_back) к указанному id имени блоку (предыдущий блок по сюжетной линии)

		"back_to": Предыдущий id блока",


"className" - Аналог атрибута className Позволяет применять стандартные классы css, указанные в разделе html документа "style".

		"className": "набор классов стилей css",


"go_to" - Переход в блок с указанным id именем, или можно указать функцианал одной из системных кнопок System Buttons

		"go_to": "Следующий id блока",


"style" - необязательный параметр (аналог атрибута style в html) персональный css стиль для этого изображения

		"style": "параметры и их значения css",


"sound" - Запускает указанный звук, Указать адрес файла нельзя только id имя из раздела "parameters" / "sounds"

		"sound": "Id звука",

Поддерживается вариант с локализацией 
	"sound": {
		"en":"Id звука для английского",
		"ja":"Id звука для японского",
		"ru":"Id звука для русского"
		}
	}


"sound_stop" - Останавливает указанный звук

		"sound_stop": "Id звука для остановки",


"position" - Расположение элемета относительно краев сцены, Например значения [0, 10px, 10px, 0] означают, что элемета появится в правом верхнем углу с отступом в 10 пикселей.

		"position": [лево, право, верх, низ] позиция относительно края сцены,


"url" - Указывает путь к файлу или иному ресурсу (аналог атрибута src в html). 

		"url": "путь к файлу",


"size" - Размер элемента. Возможно использовать пропорциональные значения относительно размера сцены. 

		"size": ["ширина", "высота"],


"size_text" - Размер текста внутри элемента

		"size_text": "Размер текста"


"color" - Цвет фона элемета в шестнадцатеричном формате hex (#fff #ffffff) или в форматах rgb (255,255,255) и rgba (255,255,255,1). По умолчанию элемета прозрачная

		"color": "цвет фона",


"color_text" - Цвет текста, необязательный параметр, если не указан то цвет будет по умолчанию, цвет можно установлен в шестнадцатеричном формате hex (#fff #ffffff) или в форматах rgb (255,255,255) и rgba (255,255,255,1).

		"color_text": "Цвет текста",


"indent_text" - Аналог css параметра padding Определяет величину отступа от краев элемента, Можно использовать несколько значений в соответствии с правилами заполнения css. Для разных отступов по вертикали и горизонтали "10px, 5px". Если нужны отступы для каждой стороны или только одной "16px, 5px, 5px, 5px"

		"indent_text": "отступы для текста",


"text" - текст для отображение внутри элемента

		"text": "текст",

Поддерживается вариант с локализацией 
	"text": {
		"en":"Перевод текста на английский",
		"ja":"Перевод текста на японский",
		"ru":"Перевод текста на русский"
		}
	}

Функции tuesday.js



load_story(tip, url) - запуск сценария истории, tip отвечает за тип сценария:

tip "file" если сценария в json файл и в значение "url" нужно указать путь к файлу

tip "data" если сценария в let массиве и в значение "url" нужно указать название массива


go_story() - двигает историю вперед по сюжету. Автоматически переходит к следующему блоку, сцене и диалогу строго по их последовательности. переход на следующий блок по параметру go_to

back_story() - Откатывает историю назад. Автоматически переходит к предыдущему блоку, сцене и диалогу строго по их последовательности. переход на рондыдущий блок по параметру back_to


save_stag(tip) - Сохранение массива текущего состояния истории, такие занижения как текущая сцена диалог блок и все значения из раздела variables в localStorage параметр "tip" будет означать название сохранения. Если указать "slot" то в localStorage он отобразится как tue_slot_data. Занятые названия bookmark для ручного сохранения и auto для автоматического сохранения.


load_stag(tip) - Загрузка массива стадии истории. Параметр "tip" название сохранения, bookmark для ручного сохранения и auto для автоматического сохранения. Или возможно использовать любое другое название которое применялось при использовании функции save_stag(tip)


go_to(go) - Переход в определенный блок истории указав в параметре go его название


full_screen() - puts the Tuesday element in full screen mode







События


save - Ручного сохранения (не работает на autosave)

tuesday.addEventListener('save', function(event) { alert("save") } );

load - Загрузка ранее сохраненной стадии прогресса

tuesday.addEventListener('load', function(event) { alert("load") } );

script_loaded - Сценарий истории загружен успешно

tuesday.addEventListener('script_loaded ', function(event) { alert("script_loaded ") } );

_add - Добавление или вычитание из определенной переменной, имя переменной с добавленным "_add"

tuesday.addEventListener('variabl_add ', function(event) { alert("change") } );

_set - Изменение определенной переменной, имя переменной с добавленным "_set"

tuesday.addEventListener('variabl_set ', function(event) { alert("set") } );

creation_dialog - Окончание создание новой сцены или диалога

tuesday.addEventListener('creation_dialog ', function(event) { alert("creation_dialog ") } );





Плагины



Touch Swipe


Позволяет переключать диалоги с помощью свайпа по сенсорному экрану. Мышь не поддерживается

 <script type="text/javaScript" src="touch_swipe.js"></script>



Toast


Не большое сообщение всплывающие на некоторое время

 <script type="text/javaScript" src="toast.js"></script>

Кастомизация css по id toast

#toast {
	background-color: rgba(74,64,69,0.7);
	border-radius: 24px;
	border: 0.1em solid rgba(207,173,156,0.7);
	height: 48px;
	width: 256px;
}

toast(message) - показывает сообщение, параметр message задает текст сообщения.


Пример использования в javaScript ( вывод текста на событие загрузки прогресса)

tuesday.addEventListener('load', function(event) {toast("load")});

Пример использования в tusday. используется в параметрах dialogs, choice и legacy_choice

"toast": " текст сообщения",
